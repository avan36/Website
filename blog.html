The Future of Software

I've been wondering recently about the future of software as generative tools have become part of how we think about work. It seems to me that, throughout the history of software companies, their moat has been time and technical know-how.

It would take many lifetimes to create the same software that Microsoft offers today, and far more knowledge than any one person could hold. That's the moat — its products are hard and time-consuming to build.

But here comes a new technology that can create virtually unlimited lines of code in no time. So the question that keeps returning to me is simple: what is the future of software if a machine can produce any code and act on its own?

Of course, it would be impractical right now to replicate something like Microsoft Word from a single prompt. How would you even go about it? It's still hard to juggle many files at once, so you'd find yourself copying and pasting back and forth between the model and the editor. Newer tools help, but they have real limits on what you can build. Even so, the trajectory is clear: the technology will get cheaper, faster, and better.

I don't mean to wax hyperbolic or glorify some inevitable "future." I'm speaking practically. And that brings me back to the point — if the code itself becomes cheap, what stays valuable?

I'm reminded of the example of a model producing a working HTML/CSS page from a single image. I've tried it myself, taking screenshots of various sites and asking for a copy — the results were genuinely impressive. Soon, you'll be able to record a few minutes of an app like Duolingo and get a working demo back, source code included.

So one irreplaceable part of software turns out to be the things that aren't code at all: the social features and the network effects built around them. Copying Facebook pixel-for-pixel from a screenshot wouldn't get you anywhere — you'd have none of the content and none of the enormous graph of people that makes it matter. The scarce thing was never the code. It was the judgment about what to build, and the people who showed up to use it.
